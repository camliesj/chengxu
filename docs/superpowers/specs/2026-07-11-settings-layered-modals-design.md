# Settings Layered Modals Design

## Goal

将系统设置中的基础字典、账号管理和操作日志从纵向展开区域改为分层弹窗，缩短页面并让管理、编辑、删除三个操作层级更清晰。

## Structure

- 系统设置首页保留四个入口卡片：保险公司字典、人员岗位字典、账号密码管理、操作日志。
- 点击入口打开一级管理弹窗，标题栏和操作区固定，中间列表独立滚动。
- 点击新增或编辑打开二级表单弹窗，保存后关闭二级弹窗并刷新一级列表。
- 点击删除打开三级确认弹窗，明确显示删除对象，确认后返回一级弹窗。
- 一级遮罩使用柔和模糊，二级和三级遮罩逐层加深，窗口层级分别使用递增 z-index。

## Behavior

- 点击遮罩或关闭按钮仅关闭当前最上层弹窗。
- 打开账号管理时刷新账号；打开日志时刷新日志。
- 字典、账号、权限和日志继续使用现有云端接口，不改变数据结构。
- 桌面端一级弹窗宽度约 1040px，二级表单约 720px；移动端接近全屏。

## Verification

- 四个入口能打开对应一级弹窗。
- 字典和账号可新增、编辑、删除，保存后列表保持可见并刷新。
- 日志可刷新且仅弹窗内容滚动。
- `npm.cmd run build` 通过，页面无横向溢出。
