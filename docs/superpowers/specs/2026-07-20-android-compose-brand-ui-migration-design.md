# Android Compose 品牌 UI 移植设计

## 背景

Android 客户端已经具备真实双公司认证、加密会话、员工/管理员权限、离线只读门禁、Navigation 3 五栏导航、工作台状态模型与可安装 APK。当前生产 Compose 界面仍以基础 Material 3 组件为主，与已经完成并通过用户评审的品牌 HTML 原型存在明显视觉差距。

本阶段把 `design/mobile-ui/` 中已经通过 Design QA 的品牌视觉和核心交互移植到 `android-client/`。HTML 原型是本阶段的视觉与交互真值；Compose 实现不得另行发明颜色、布局语言、图标风格或页面信息架构。

视觉真值与资产来源：

- `design/mobile-ui/reference/xpeng-ui-overview.png`
- `design/mobile-ui/reference/xpeng-ui-interaction-states.png`
- `design/mobile-ui/output/brand-prototype-*.png`
- `design/mobile-ui/public/brand-assets/login-service-vehicle.png`
- `design/mobile-ui/public/brand-assets/empty-service-tools.png`
- `design/mobile-ui/src/tokens.css`

真实工单 API 与 Room 缓存计划继续暂停，直到本阶段完成并交付真机 APK。

## 目标与成功标准

完整升级当前已经可用的 Android 界面：

- 恢复中根界面；
- 登录、双公司选择、账号密码、密码显隐、错误与提交中状态；
- 员工工作台；
- 管理员工作台；
- Navigation 3 五栏壳层；
- “我的”、身份信息与退出确认；
- 离线只读条幅和所有相关禁用状态；
- 工单、新增、档案的品牌化阶段页与空状态。

成功标准：

1. Compose 版在 390×844 视觉基准下保持 HTML 原型的层级、色彩、圆角、间距、图片裁切、导航结构和主要交互反馈。
2. 360dp 窄屏无横向滚动、文本或主操作裁切；更高屏幕由内容区滚动和固定底栏自然适配。
3. 认证、会话、权限、离线门禁、Navigation 3 返回栈和工作台状态计算的业务语义不变。
4. 员工不能看到管理员结算入口；离线时所有写操作继续被业务门禁拒绝，中央“新增”明确禁用。
5. JVM 单元测试、Android 测试代码编译、`lintDebug` 和 Debug APK 构建全部通过。
6. 不启动 Android 模拟器；最终 APK 放入发布目录，由用户在真实手机上完成视觉与触控验收。

## 已选迁移方式

采用“组件优先、完整页面纵向迁移”。先锁定设计系统、图标与图片资产，再依次迁移登录、五栏壳层与个人页、双角色工作台，最后进行全量验证和 APK 交付。

不采用一次性大替换，因为它会把认证、导航和视觉回归集中到同一批改动；不采用新旧 UI 双轨功能开关，因为它会产生两套界面、重复测试和长期清理成本。纵向迁移不会缩减最终效果，每个阶段都能独立测试、提交、推送和回退。

## 架构与状态边界

### 保留的生产边界

- `AuthenticationRepository` 继续拥有恢复、登录、过期和退出会话的生命周期。
- `LoginViewModel` 继续拥有公司、账号、密码、提交中和错误状态；品牌登录页不直接调用 API。
- `WorkbenchViewModel` 继续组合会话、网络和工作台仓库，并生成 `WorkbenchUiState`。
- `MutationGate` 和 `MutationDecision` 继续是写操作的唯一权限与离线判断来源。
- `AppNavigationState`、`RootTab` 和 `NavDisplay` 继续管理五个独立根返回栈。
- Android 不加入员工/管理员调试切换器；角色只来自真实认证会话。

### UI 数据流

```text
AuthenticationRepository / NetworkMonitor / WorkbenchRepository
                         ↓
             LoginViewModel / WorkbenchViewModel
                         ↓
             LoginUiState / WorkbenchUiState
                         ↓
          品牌 Compose 页面与共享组件
                         ↓
       既有回调、导航事件与权限门禁入口
```

品牌组件只接收稳定数据和显式回调，不读取仓库、不推断角色、不绕过门禁。允许的工作台快捷操作在本阶段只能进入已经存在的阶段页或给出诚实的未开放提示；不得伪造创建、结算、保存或同步成功。

## 品牌设计系统

### 颜色

Compose Token 与 HTML 原型保持一致：

- Canvas：`#F4F6F8`
- Surface：`#FFFFFF`
- Surface Soft：`#F0F3F7`
- Ice：`#EAF1FB`
- Ink：`#101214`
- Ink Muted：`#697079`
- Line：`#E3E7EC`
- Action：`#111315`
- Action On：`#FFFFFF`
- Success：`#25805F`
- Warning：`#A96816`
- Danger：`#B84A45`

Material 3 色槽只能映射到这些品牌 Token，不允许页面直接写入新颜色。语义色通过受限枚举提供，组件 API 不公开任意 `Color` 参数，延续原 Task 4 的色板契约防线。

### 尺寸与排版

- 页面水平留白：16dp；主要区块间距：20–24dp；密集内容间距：8–16dp。
- 页面或弹层大容器：20dp 圆角；卡片：16dp；输入框和按钮按 HTML 原型映射为 14–16dp；圆形按钮保持完整圆形。
- 最小触控目标：48×48dp。
- 使用 Android 中文系统字体、零额外字距；通过明确的品牌 Typography 锁定标题、数字、正文和标签层级。
- 内容阴影轻量、只用于需要层级的悬浮面板；普通列表优先使用表面色和纤细分隔线。
- 动效以 120–180ms 为主；系统请求减少动态效果时取消非必要缩放和位移。

### 共享组件

设计系统提供以下受控组件：

- 品牌主按钮、次按钮、安静按钮和危险按钮；
- 品牌输入框与密码尾部动作；
- 公司选择卡；
- 状态标签；
- 指标卡；
- 工单卡；
- 导航项和中央新增入口；
- 离线条幅；
- 居中确认弹窗；
- 空状态面板。

组件必须覆盖默认、触摸按下、键盘/辅助输入焦点、禁用和选中状态；存在鼠标或触控板时增加悬停反馈。选中和禁用不能只依赖颜色，必须同时改变容器、图标、文字权重或禁用语义。所有可点击卡片使用真实点击语义和明确的无障碍描述。

## 图标与图片资产

### Hugeicons

HTML 原型已经使用 Hugeicons Stroke Rounded。Compose 没有官方运行时包，因此把原型实际使用的 Hugeicons 官方 SVG 转换为本地 Android VectorDrawable，通过单一图标入口引用。

要求：

- 只转换实际消费的图标；
- 保留原始图标名、来源包版本和许可记录；
- 统一使用接近 1.5px 的 Stroke Rounded 视觉；
- 不混入 Material Icons、自制 SVG、Emoji 或文字字形；
- 应用运行时不请求图标服务，也不包含授权密钥。

### 品牌图片

复用已经生成并通过透明边缘检查的：

- `login-service-vehicle.png`：登录主视觉；
- `empty-service-tools.png`：工单、新增和档案阶段页空状态。

图片进入 Android drawable 资源时保持透明通道、主体比例和原型裁切，不重新生成、不拉伸。按消费槽位确定 `ContentScale`、对齐和最大高度，并检查 360dp 与 390dp 下均无主体截断。

## 页面设计

### 恢复中

恢复中使用品牌画布、克制的进度指示和无敏感信息文案。恢复完成前不渲染登录表单、姓名、公司或上一会话工作台，避免会话内容闪现。

### 登录

上部为浅冰蓝氛围与车辆主视觉，下部为圆角白色登录面板。面板包含标题、两张企业选择卡、账号字段、密码字段、密码显隐、字段/服务端错误和近黑色主按钮。

保留现有真实登录逻辑：空输入校验、离线拒绝、重复提交保护、提交中禁用、成功后清空密码。企业选中状态同时呈现边框/容器、图标和语义状态。软键盘出现时页面可滚动，当前字段和登录按钮不得被永久遮挡。

### 五栏壳层

底栏顺序继续固定为“工作台 / 工单 / 新增 / 档案 / 我的”。底栏使用白色低干扰表面和纤细顶部分隔线；当前项通过图标、文字权重和容器共同表达。中央“新增”采用近黑色圆形入口，离线时原生禁用并暴露禁用语义。

主内容区独立滚动，底栏固定。重复选择当前标签继续只重置该标签返回栈，切换标签继续保留其他标签的独立栈。

### 员工与管理员工作台

两个角色共享同一 `BrandWorkbench` 组合和组件体系，只消费不同的 `WorkbenchUiState`。

- 顶部：问候、公司、网络状态和轻量个人入口。
- 摘要带：四项核心状态，窄屏通过缩小间距和数字字号适配，不使用横向滚动。
- 指标：员工显示今日接车、在修车辆、待交付、保险到期；管理员显示本月产值、待结算金额、在修车辆、保险到期。
- 快捷操作：只显示状态模型提供的动作；管理员才有结算入口。
- 近期工单：整卡可聚焦和点击，显示车牌/客户、维修摘要、状态、工单号和金额。

拒绝操作继续显示 `MutationDecision.Denied.reason`。UI 不根据角色名称二次推断动作权限。

### 工单、新增和档案阶段页

三页复用同一空状态骨架，包含透明维修工具图片、阶段标签、明确标题、简短说明和一个非破坏性操作。文案明确真实数据或业务表单将在后续阶段接入；不得显示可造成写入成功错觉的表单或按钮。

离线时“新增”标签及其写入口保持禁用；工单和档案阶段说明仍可查看。

### 我的与退出

“我的”显示身份卡、当前企业、角色、同步状态、安全提示和退出按钮。退出使用居中白色确认弹窗和深色半透明遮罩；初始焦点位于安全的取消动作，返回键/取消关闭弹窗，关闭后焦点回到退出按钮。确认后调用现有退出逻辑，清除加密会话并回到登录页。

### 离线

壳层顶部继续显示精确文案“网络不可用，当前为只读模式”。所有写操作由既有门禁禁用或拒绝，只读内容、根标签切换和资料查看仍可用。视觉层不得仅靠降低透明度伪装禁用，必须使用真实 `enabled = false` 或明确的拒绝事件路径。

## 错误处理与无障碍

- 登录字段错误靠近对应字段，服务端或存储错误放在表单动作附近，并使用现有安全文案。
- 加载期间阻止重复提交，但不清除用户已经输入的账号和公司。
- Snackbar 只承载短期动作反馈；需要决策的危险操作使用确认弹窗。
- 弹窗具有明确标题、描述、确认/取消语义，并支持系统返回键关闭。
- 内容描述面向动作结果而不是图标外形；装饰图标不重复朗读。
- 文字和关键图标满足可读对比度；语义状态不只通过颜色区分。

## 测试与验证

采用 TDD，小步提交。测试层次如下：

1. JVM 单元测试锁定品牌色值、Material 3 色槽映射、圆角/间距合同和无任意颜色逃逸。
2. Compose Android 测试覆盖登录企业选择、密码显隐、提交/禁用、五栏顺序与选中、离线新增禁用、员工/管理员内容差异、退出弹窗与返回登录。
3. 既有 `LoginViewModelTest`、`WorkbenchViewModelTest`、`AppNavigationStateTest` 和会话/权限测试保持通过，证明视觉迁移未改变业务规则。
4. 每个重要阶段至少运行相关 JVM 测试和 Android 测试代码编译；风险较大的导航、认证和最终阶段增加 `lintDebug`。
5. 最终执行干净构建：

```powershell
.\gradlew.bat clean :app:testDebugUnitTest :app:compileDebugAndroidTestKotlin :app:lintDebug :app:assembleDebug
```

6. 将 Debug APK 复制到 `dist/releases/android/autoservice-android-debug-0.1.0.apk`，记录大小和 SHA-256。
7. 不启动 Android 模拟器。用户在真实手机上验证登录、员工/管理员工作台、五栏导航、离线、退出、360dp 级窄屏适配和触控反馈；发现视觉差异后进行一次定向微调。

## 迁移阶段与提交边界

1. 品牌 Token、Typography、形状、共享组件、Hugeicons VectorDrawable 和两张图片资产。
2. 恢复中、登录、双公司选择与认证状态。
3. 五栏壳层、离线条幅、三张阶段页、“我的”和退出弹窗。
4. 员工/管理员工作台、指标、快捷操作和近期工单。
5. 全量回归、Lint、APK、发布文档与真机清单。

每个阶段完成后必须更新 `docs/latest-handoff-prompt.md`、提交 Git 并推送 `codex/android-mobile-ui-atlas`。除非用户重新授权，过程中不启动 Android 模拟器。

## 非目标

- 不实现真实工单 API、Room 缓存、工单新增/编辑、结算或档案业务。
- 不改变服务端接口、认证存储格式、角色权限或离线门禁规则。
- 不把 HTML 原型的员工/管理员调试切换器带入生产 Android。
- 不复制参考产品品牌标志、车型、社区、资讯、商城或 iOS 系统界面。
- 不维护新旧两套生产 UI，不加入长期功能开关。
- 不使用模拟器作为本阶段验收条件。

## 设计决策总结

本设计把已经通过评审的 HTML 原型作为唯一视觉真值，通过共享品牌组件逐页替换基础 Material 3 组合，同时严格保留现有业务与状态边界。迁移的风险通过纵向阶段、TDD、小步提交和最终真机 APK 验收控制；视觉升级不会提前扩张工单业务范围。
