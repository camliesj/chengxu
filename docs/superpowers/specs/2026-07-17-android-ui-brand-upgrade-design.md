# Android 品牌化 UI 升级设计

## 背景与优先级

Android 生产客户端已具备真实登录、员工/管理员工作台、Navigation 3 五栏壳层、“我的”与离线只读状态，但当前视觉仍接近基础 Material 3 组件组合。用户要求先进行 UI 升级，再实施已经设计完成的真实工单 API 与 Room 缓存。

本阶段先升级 `design/mobile-ui/` HTML 原型并完成视觉、交互和自动化验收；原型获批后再单独制定 Jetpack Compose 移植计划。真实工单缓存设计与计划保留但暂停执行。

参考图来自汽车品牌原生应用界面合集。项目借鉴其视觉语言和交互质感，不复制品牌标志、车型名称、资讯内容或 iOS 系统界面。

### 参考图特征拆解

- 大面积白色、雾灰、浅冰蓝背景，近黑色作为高权重操作色；鲜艳颜色只承担局部强调。
- 页面上方留白充足，标题字重高、正文对比度低，信息通过间距和字重而非密集边框分组。
- 车辆或内容图片承担氛围入口，正文和操作承载在圆角白色面板、卡片或底部面板中。
- 卡片圆角柔和、阴影极轻、分隔线纤细；全局不使用厚重描边或高饱和科技蓝堆叠。
- 弹窗使用深色半透明遮罩与高对比白色容器；主次按钮通常采用黑/白反差。
- 导航、标签、输入框和卡片交互反馈克制，依靠色阶、透明度、轻微缩放和位移表达。

## 已确认范围

完整升级当前已可用的界面：

- 登录与双公司选择；
- 员工工作台；
- 管理员工作台；
- Navigation 3 五栏底部导航；
- “我的”与退出登录；
- 恢复中、离线、错误、禁用、弹窗和底部面板等系统状态。

“工单 / 新增 / 档案”只同步新的导航壳层、页面标题和高品质空状态，不提前实现真实业务表单或写入能力。

## 设计方向

采用“原生高保真转译”：保留参考图的汽车品牌应用气质、空间节奏与交互反馈，但信息架构继续服务汽修接待和经营管理。不会照搬参考产品的资讯流、黑色启动页、车型商城或社区结构。

允许替换现有 Android 亮蓝视觉。新的基础方向为：

- 页面底色：白、雾灰与浅冰蓝氛围层；
- 主文字与主操作：近黑色；
- 次级文字：中性灰；
- 内容表面：白色或半透明浅色表面；
- 业务语义：在修使用低饱和绿、待结算使用琥珀、危险/过期使用低饱和红、普通信息使用冰蓝；
- 大圆角容器 16–20，内部卡片 12–16，圆形图标按钮保持完整圆形；
- 主要内容水平留白 16，区块间距 20–24，密集列表内部间距 12–16；
- 动效时长 120–220ms，支持系统减少动态效果。

最终色值、字阶、圆角和阴影必须在 390×844 HTML 原型视觉复核中锁定，Compose 不得自行二次发明。

## 页面设计

### 登录与公司选择

页面上半部为浅冰蓝氛围背景与汽修车辆/车间主视觉，使用真实生成或项目已有图片，不使用插画占位框。下半部为悬浮白色登录面板，包含品牌化标题、公司选择、账号、密码、显示密码、错误说明和近黑色主按钮。

公司选择采用可清晰比较的两张选择卡或紧凑选择面板；选中状态不能只依赖颜色，必须同时有图标与边框/容器变化。登录原型使用模拟状态，不连接真实 API，但空输入、错误、提交中、禁用和成功跳转均可操作。

### 员工工作台

渐变头部包含问候、公司、在线/离线状态和轻量个人入口。首屏优先展示横向工单状态摘要与四张业务指标卡，随后是快捷动作、当班信息和近期工单。信息密度保持工作效率，不采用大面积资讯流图片挤压业务内容。

### 管理员工作台

与员工工作台共享骨架和组件。经营摘要优先展示本月产值、待结算金额、在修车辆和保险到期；高优先事项与近期工单置于其后。管理员身份通过内容和权限体现，不另造一套视觉系统。

### 五栏导航与占位页

底部为低干扰白色导航栏，包含“工作台 / 工单 / 新增 / 档案 / 我的”。中间“新增”采用近黑色圆形主入口。激活项通过容器、图标色与文字字重共同表达。

工单、新增和档案占位页使用同一头部、导航和空状态体系；空状态包含透明背景的车辆/工具主题图片、明确说明和不可误导的后续阶段提示，不伪造可写业务。

### 我的与系统状态

“我的”包含身份卡、所属门店、角色、同步状态、安全提示和退出登录。退出使用确认弹窗，危险操作保持清晰但不使用大面积刺眼红色。

确认类操作使用居中圆角白色对话框；选择和筛选使用圆角底部面板。离线条幅继续显示“网络不可用，当前为只读模式”，并禁止所有写操作。

## 组件与交互状态

所有可操作组件，包括按钮、图标按钮、导航项、选择卡、输入框、指标卡、工单卡、标签、弹窗动作和底部面板选项，必须实现并展示：

- 默认：标准表面、文字和轮廓；
- 悬停：仅在支持指针的设备上轻微提亮、抬升或显示柔和色阶；
- 按下：约 98% 缩放与短促色阶反馈，不能引发布局跳动；
- 禁用：降低内容与容器对比度，取消阴影和交互，暴露禁用语义；
- 键盘焦点：清晰可见的焦点环，不与悬停状态混淆；
- 选中：图标、文字和容器同时变化，不能只依赖颜色。

触控目标不小于 48×48。键盘可完成登录、导航、打开/关闭弹层和确认/取消。关闭弹层后焦点返回触发控件。减少动态效果开启时取消非必要位移和缩放。

## Hugeicons 策略

默认采用 Hugeicons Stroke Rounded，常规视觉相当于 1.5px 描边。Hugeicons 官方目前提供 Web、React、React Native、Flutter、SVG/Icon Font 等形式，没有官方 Jetpack Compose 包，因此分两阶段处理：

1. HTML 原型使用 Hugeicons 官方免费包或官方导出的真实图标资源；只导入使用到的图标。
2. Compose 移植时，将已确认的 Hugeicons SVG 资源转换为本地 Android VectorDrawable，并保留来源与许可记录；应用运行时不请求图标 API，不提交授权密钥。

不得手绘、近似临摹或混入其他图标库。若某个业务概念没有合适 Hugeicons，必须先重新选择最接近的 Hugeicons，不以文字字符、Emoji 或自制 SVG 替代。

## 图片资产策略

先盘点项目现有图片和参考图中需要的资产槽位，再为确实缺失的内容调用内置图像生成器。预期槽位包括登录主视觉和占位/空状态车辆或工具素材。

每个资产生成前必须确定消费组件、显示尺寸、裁切方式、焦点、色温和背景要求。需要透明底时生成主体并完成透明背景处理，导出 PNG 后检查边缘、阴影、缩放和深浅背景适配。不得从参考合集粗暴裁切含水印、文字或其他品牌标志的图片。

## HTML 原型实现与验证

升级现有 `design/mobile-ui/` Vite/React 原型，不新建另一套重复工程。原型保持 390×844 基准画布，并检查更窄手机和较宽窗口无横向滚动、裁切或离屏主操作。

核心交互必须真实工作：

- 公司选择、账号密码输入、密码显隐、校验、模拟登录和退出；
- 员工/管理员调试切换；
- 五栏导航和当前项；
- 快捷动作、指标卡和工单卡反馈；
- 弹窗与底部面板开关、确认、取消和遮罩关闭；
- 状态画廊集中展示默认、悬停、按下、焦点、选中、加载和禁用状态。

自动化覆盖页面目录、路由、登录交互、角色切换、五栏导航、弹层开关、禁用行为和状态画廊。视觉测试以 390×844 截图为基准。

设计 QA 必须把参考图与同尺寸原型截图并排比较，检查布局、边距、字重、圆角、图像裁切、颜色和交互状态。所有 P0、P1、P2 差异必须修复；P3 仅可作为后续微调记录。HTML 原型未通过视觉验收前不进入 Compose 实现。

## Android 移植边界

HTML 原型通过用户评审后，另行编写 Compose 移植实施计划。移植必须复用当前权限、会话、Navigation 3、离线门禁与工作台状态模型，不改变认证和业务规则。

Android 验证继续包含 JVM 单元测试、Android 测试代码编译、`lintDebug` 和 Debug APK 构建；按照用户决定不启动模拟器，APK 交由真实手机验证。

## 非目标

- 不复制小鹏品牌标志、车型内容、社区、资讯或商城功能；
- 不在 HTML 原型中连接生产 API、保存真实凭据或实现服务端认证；
- 不在本阶段实现工单新增、编辑、结算或档案业务；
- 不执行已规划的 Room 工单缓存任务，直到 UI 升级和 Compose 移植完成或用户重新调整优先级；
- 不以自制图形、Emoji、占位框或其他图标库代替缺失资产。
