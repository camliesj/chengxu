# Android 紧凑自适应登录页设计

## 背景

当前 Android 登录页在常见手机上纵向内容过长：330dp 的 Hero、三行重复登录说明、两张 72dp 企业卡以及较大的表单间距叠加后，账号、密码和主按钮落到首屏之外；白色表单面板还过度覆盖车辆主视觉。用户已经选择方案 A：保留现有品牌 Hero 和纵向企业选择，但压缩信息层级，并在输入法弹出时进一步收拢顶部区域。

本次只调整 Android 原生 Compose 登录页，不修改认证、会话、公司 ID、错误映射或网络逻辑，也不修改网页端。

## 目标与验收基线

- 以 360×800dp 为最低常规视口：未弹出输入法、系统默认字体缩放时，无需滚动即可看到完整的“进入系统”按钮。
- 390×844dp 等常见视口应同时看到按钮下方的安全连接说明，并保留可辨识的车辆主视觉。
- 企业卡、输入框、密码可见性按钮和主按钮的有效触控高度不得小于 48dp。
- 键盘弹出后自动切换为更紧凑的 Hero，尽量让当前输入框与主按钮留在可见区；极小屏幕、大字号和特殊系统栏组合仍保留纵向滚动作为无障碍兜底。
- 现有默认、选中、按下、禁用、加载和错误状态保持可交互并沿用品牌设计系统。

## 信息层级与文案

### 常规 Hero

- 高度固定为 200dp，继续使用现有冰蓝背景和 `brand_login_service_vehicle` 车辆资产。
- 保留品牌眉题 `AUTOSERVICE MOBILE`。
- 主标题保留“让每一次服务更从容”，使用两行紧凑排版和较现状更小的标题字号。
- 删除 Hero 内“门店业务与车辆服务协同入口”，避免与表单说明重复。
- 车辆图片固定在 Hero 底部；源 PNG 上下包含较大透明区，因此保留 154dp 图片槽以获得约 60dp 的可见车辆主体，左右保留 20dp 安全边距。主体应结束在面板覆盖区域上方，不得被表单面板大面积遮挡。

### 表单面板

- 面板只向上覆盖 Hero 16dp，圆角与颜色沿用现有 `AutoservicePanelShape` 和品牌色。
- 内边距采用水平 16dp、垂直 16dp；主要内容间距为 10–12dp。
- 删除“欢迎回来”眉题，仅保留“登录维修业务移动端”和一行辅助文案“选择企业并使用业务账号登录”。
- 保留“选择企业”标签。
- 两家企业继续纵向排列，以避免中文名称在窄屏中被强制截断；每张卡高度压缩到 56dp，只显示短业务名称：
  - 通达汽车服务中心
  - 鑫齐恒汽车服务中心
- 登录页不再展示完整工商登记名称；公司 ID 和提交值保持不变。
- 账号、密码字段保持现有输入语义、IME 动作、密码显隐和禁用状态。
- 错误文案仅在失败时占位；无错误时不预留空白高度。
- “进入系统”按钮保持全宽和至少 48dp 高度。
- 安全说明改为单行水平组合：盾牌图标 + “账号凭据将通过安全连接提交”，减少尾部高度。

## 键盘与小屏自适应

- 使用 Compose `WindowInsets.ime` 判断输入法是否可见，不依赖设备型号或像素阈值。
- 输入法可见时，Hero 切换为约 96dp 的紧凑状态：保留品牌眉题与单行登录上下文，隐藏车辆图片和非必要宣传文案。
- 页面继续使用 `imePadding()`，避免主按钮被键盘直接覆盖。
- 常规 360×800dp 布局不依赖滚动；`verticalScroll` 仅作为输入法、极小高度、横屏或系统大字号下的安全兜底，因此不会牺牲可达性。
- 尺寸变化不清空账号、密码、企业选择、错误或提交状态。

## 组件与交互契约

- 企业选择继续复用 `CompanySelectionCard`，为其增加紧凑呈现能力，而不是复制一套登录页专用卡片。
- 紧凑企业卡仍使用本地 Hugeicons 建筑与选中图标；默认态、选中态、按下态和提交中的禁用态沿用现有颜色与边框反馈。
- `CompanySelectionCard` 的默认尺寸和其他调用方不变；只有登录页显式启用紧凑模式并省略 supporting text。
- `BrandTextField`、`BrandButton` 和密码显隐 `IconButton` 继续复用现有组件，不改变对外业务回调。
- 点击键盘 Done 仍调用登录；提交中继续阻止企业切换、输入修改、密码显隐和重复提交。

## 测试驱动与验证

先写失败契约，再补最小实现：

1. 更新 360×800dp Compose 测试，直接断言登录按钮已显示，不再调用 `performScrollTo()`。
2. 增加常规态 Hero、两张紧凑企业卡和主按钮的语义标签/尺寸契约；企业卡与主按钮触控高度至少 48dp。
3. 保留企业切换、密码显隐、表单校验、错误、加载和禁用相关既有测试。
4. 为键盘紧凑状态抽取可测试的布局决策或稳定语义，锁定“IME 可见时使用紧凑 Hero”，避免设备型号分支。
5. 执行 Android JVM 单元测试、Android 测试代码编译、`lintDebug` 和 Debug APK 构建；按用户要求不启动模拟器、不声称连接式 UI 测试已运行。
6. 生成并哈希新的可安装 APK，交由用户在真实手机验证 360dp 宽度、系统字体缩放、输入法遮挡和两家企业真实登录。

## 非目标

- 不更换现有车辆图片、品牌配色、字体体系或 Hugeicons 图标来源。
- 不新增验证码、记住密码、第三方登录、企业搜索或登录方式。
- 不改变服务端 API、生产地址、认证凭据处理、Keystore 存储或 Room 工单缓存。
- 不在本阶段启动 Android 模拟器或执行连接式设备测试。
