# Windows PC 客户端设计

## 1. 目标与范围

在现有 React + Vite 汽修管理系统基础上增加 Windows PC 客户端。PC 客户端复用网页端业务页面、Cloudflare Pages Functions、D1 数据库和腾讯云 COS，不另建一套业务系统。

首版仅支持 Windows 10/11。应用必须联网才能执行写操作；不实现离线新增、离线编辑、待同步队列或冲突合并。网页端与 PC 端共用源码，但独立发布版本：网页发布不会立即改变已安装客户端，用户完成 PC 应用更新后才切换到新版本。

## 2. 技术方案

采用 Tauri 2 封装现有 React + Vite 前端，生成 Windows NSIS 安装包。

- React 继续承载全部业务界面与状态逻辑。
- `src-tauri` 只负责窗口、单实例、网络请求、系统对话框、打印、下载和自动更新等桌面能力。
- Windows 使用系统 WebView2，不随安装包重复携带完整 Chromium。
- PC 应用静态资源由安装包内置，避免网页部署即时改变门店客户端。
- Cloudflare D1 是唯一结构化业务数据源，腾讯云 COS 继续保存到账回执。

不采用 Electron，原因是当前业务不需要独立 Chromium 和 Node.js 运行时，其安装体积和内存成本高于必要水平。不采用 PWA，原因是其版本固定、原生下载、打印和自动更新能力不符合已确认的发布策略。

## 3. 项目结构与共享边界

项目保持单仓库：

- `src/`：网页端和 PC 端共享的 React 页面、业务逻辑和样式。
- `src/platform/`：平台适配层，封装 API 请求、文件保存、打印、外部链接、网络状态和更新能力。
- `src-tauri/`：Tauri 配置、Rust 入口、能力权限、Windows 安装包和更新配置。
- `functions/`、`migrations/`：继续作为 Cloudflare 后端，不嵌入 PC 安装包。

业务组件不能直接依赖 Tauri API。网页端和 PC 端通过同一套平台接口调用各自实现，以便后续业务功能只修改一次。

## 4. API 与数据流

网页端继续使用同源 `/api/*`。PC 端通过统一 `apiClient` 将相同路径映射到固定生产地址 `https://chengxu.pages.dev/api/*`。

PC 端使用 Tauri HTTP 插件请求云端，能力配置只允许访问指定 HTTPS 域名。账号会话仍使用 Bearer Token，沿用当前 12 小时有效期。账号密码不保存在本机；退出登录时清除会话。

COS 密钥、D1 配置和其他服务端密钥只能保留在 Cloudflare 环境变量中，不能写入 React 构建产物、Tauri 配置或安装包。

当前保险档案和客户车辆仍存在浏览器 `localStorage` 数据源，必须在桌面封装前迁移到 D1。实施时为两类数据增加公司隔离的 D1 表和云端接口，`localStorage` 只保留最后一次成功读取的只读缓存。已有浏览器数据先复制到公司级本地备份，再由管理员在明确确认弹窗中执行一次性导入；导入使用“不覆盖同公司同 ID 云端记录”的规则，只有云端导入成功后才删除备份。内置演示保险和客户车辆数据同时移除。

## 5. 网络状态与离线行为

增加无需登录的轻量健康检查接口，用于确认业务 API 实际可用，不能只依赖 `navigator.onLine`。

- 在线时不显示额外状态条。
- 健康检查或业务请求确认离线后，顶部显示固定红色状态条：`网络不可用，当前内容为上次同步结果`。
- 保留最后一次成功读取的数据供只读查看，并显示最后同步时间。
- 新增、编辑、结算、返结算、上传、删除、导出及其他依赖云端的写操作全部禁用。
- 请求执行中断网时保留用户已填写内容，不清空表单。
- 网络恢复后重新检查健康状态，并提供重新加载云端数据的入口。
- 不保存待提交操作，不在恢复联网后自动补交业务请求。

## 6. 窗口与桌面交互

- 应用名称为`汽修接待与车辆保险管理`，两个公司共用同一安装包。
- 使用中性的车辆应用图标。
- 默认最大化启动，最低窗口尺寸为 1280x720。
- 记住上次窗口尺寸、位置和最大化状态。
- 保留 Windows 原生标题栏、最小化、最大化和关闭按钮。
- 启用单实例；重复启动时激活已有窗口。
- 关闭主窗口即退出应用。
- 首版不提供托盘驻留、开机启动、后台常驻和多窗口。
- 外部链接使用系统默认浏览器打开。

## 7. 业务能力适配

PC 客户端包含网页端现有全部业务能力：

- 首页看板
- 维修接待与工单状态流转
- 已结算历史档案
- 车辆保险
- 客户车辆
- 汇总报表
- 管理员数据导出
- 系统设置、字典、账号权限和操作日志
- 结算回执上传、查看和删除

桌面增强只放在系统能力边界：

- 工单打印调用 Windows 系统打印窗口，继续使用现有单页打印模板。
- Excel 导出调用 Windows 保存对话框，由用户选择文件名和位置。
- 回执上传调用 Windows 文件选择器。
- 回执查看继续使用应用内弹窗。

## 8. 客户端下载入口

网页端不新增导航项或独立下载页面。登录页底部增加低权重的`客户端下载`文字按钮，未登录用户也能访问。

点击后打开居中弹窗：

- Windows 卡片显示版本号、更新时间、文件大小、更新说明和下载按钮。
- Android 卡片在未发布时显示`敬请期待`，以后可由同一份版本元数据自动切换为下载按钮。
- 弹窗从云端读取版本元数据，不在 React 页面写死安装包地址。

已登录管理界面不增加侧边栏入口。PC 应用的`系统设置 -> 关于与更新`只负责查看当前版本和更新已安装客户端。

## 9. 安装包与自动更新

Windows 使用 NSIS 按当前用户安装，默认不要求管理员权限。安装时创建桌面和开始菜单快捷方式，并检测 WebView2；缺失时引导安装。

发布文件包括：

- NSIS 安装包
- Tauri 更新签名文件
- 版本元数据
- 更新说明

安装包和更新文件发布到专用腾讯云 COS 发布位置。客户回执继续保持私有；发布文件不得通过放宽整个回执存储桶权限来公开。正式发布应使用独立发布桶，或使用只对发布路径生效的严格存储桶策略。

版本元数据由 Cloudflare 通过 HTTPS 提供，同时服务登录页下载弹窗和 Tauri 更新器。元数据包含 Windows 版本、发布时间、文件大小、更新说明、下载地址、更新地址和签名，并预留 Android 字段。

更新行为：

- 应用启动后检查一次更新。
- 长时间运行期间每天最多自动检查一次。
- `关于与更新`支持手动检查。
- 发现新版后提示用户，不中断当前业务。
- 用户确认后后台下载，完成后显示`更新并重启`。
- 下载中断允许重试。
- 签名验证失败时拒绝安装。
- 更新检查失败不影响业务使用，也不反复弹窗。

发布版本标签使用 `desktop-v<major>.<minor>.<patch>`。Tauri 更新签名从首版启用，私钥必须保存在安全的发布环境中，不能提交 Git；公钥写入应用配置。丢失私钥会导致已安装客户端无法验证后续更新，因此必须独立备份。

内部测试可以使用未进行 Windows 代码签名的安装包。正式对外发布前建议购买 Windows 代码签名证书，以减少`未知发布者`和 SmartScreen 警告。Windows 代码签名与 Tauri 更新签名是两套不同机制。

## 10. 错误处理

- 会话过期：清除失效会话并返回登录界面，提示重新登录。
- 云端读取失败：保留最后一次数据，显示明确错误和重试入口。
- 云端写入失败：保留表单内容，不提前改变本地业务状态。
- 文件选择取消：保持原页面状态，不显示错误。
- 导出失败：提示保存失败，不删除已有文件。
- 打印取消：返回原页面，不改变工单。
- 更新失败：保留当前可运行版本，允许稍后重试。
- 更新签名失败：停止安装并显示安全校验失败。

## 11. 安全边界

- Tauri 权限采用最小授权，只开放指定云端 API、更新、对话框、文件保存、打印、外部链接和窗口状态能力。
- 不开放任意 Shell 命令执行。
- 不开放任意文件系统目录读写。
- 下载和保存只能由用户明确触发。
- 生产 API 只允许 HTTPS。
- 前后端继续执行现有管理员、员工和模块权限校验；桌面端禁用按钮不能替代后端鉴权。

## 12. 测试与验收

自动化测试：

- 保持现有 React 和业务逻辑测试全部通过。
- 增加平台识别、API 地址映射、网络状态机、断网写入拦截、版本比较和更新提示测试。
- 对健康检查和版本元数据接口增加 Functions 测试。
- 执行 Rust 格式检查、静态检查和 Tauri Windows 构建验证。

Windows 实机验收：

- Windows 10 与 Windows 11 安装、卸载、快捷方式和单实例行为。
- 默认最大化、最小尺寸和窗口状态恢复。
- 两家公司登录和 12 小时会话。
- 工单新增、编辑、完工、结算和返结算。
- 历史档案、保险、客户车辆、报表和管理员导出。
- 回执文件选择、上传、查看和删除。
- Excel 保存位置选择和单页工单打印。
- 断网提示、只读旧数据、写操作禁用和恢复联网刷新。
- 从旧版本发现新版、下载、验证、安装并重启。
- 登录页下载弹窗、Windows 安装包下载和 Android 未发布状态。

## 13. 明确不在首版范围内

- macOS 和 Linux
- 离线数据库与自动同步
- 系统托盘和开机启动
- 多窗口
- 后台常驻
- 静默强制更新
- Microsoft Store 发布
- Android 应用实现

Android 字段和下载展示仅作为后续兼容预留，不在本次 Windows PC 客户端实施范围内。
