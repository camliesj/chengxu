# Android 认证与会话接入设计

日期：2026-07-17  
状态：已获设计确认，待用户审阅书面规格后制定实施计划。

## 目标与范围

为独立 Android 客户端接入现有生产认证接口，使用户以“公司 + 账号 + 密码”登录，并在服务端 12 小时会话有效期内保持登录。认证成功后，现有五栏导航壳层和工作台必须使用服务端返回的角色与细粒度权限。

本阶段不接入旧版访问码登录、不新增服务端刷新令牌接口、不替换演示工单仓库，也不启动 Android 模拟器。

## 已确认的服务端契约

- 生产 API 基址：`https://chengxu.pages.dev`。
- `POST /api/access` 接收 `companyId`、`username`、`password`，成功响应为 `{ session }`。
- `session` 包含 `token`、`role`（`admin` 或 `staff`）、`label`、`companyId`、`username`、`displayName` 与 `permissions`。
- 服务端签发的 Bearer Token 有效期为 12 小时；受保护请求在 Token 缺失时返回 `UNAUTHORIZED`，失效时返回 `SESSION_EXPIRED`，均使用 HTTP 401。
- 公司的固定标识为 `tongda`（通达汽车服务中心）和 `xinqiheng`（鑫齐恒汽车服务中心）。管理员可选择任一公司；员工必须选择其所属公司。

## 方案选择

选择 Android 直连既有 HTTPS API，并以 Android 构建配置提供生产基址。这个方案复用已有认证、权限与会话表，不改变网页端或服务端协议。

不选择“每次启动重新登录”，因为会增加一线人员的重复操作；也不在本阶段新增刷新令牌机制，因为现有 12 小时服务端会话已能满足当前生产基础阶段。

## 架构与职责

### 网络层

- `AuthApi` 仅负责将登录请求序列化为 JSON 并解析成功或失败响应。
- `AuthRepository` 协调登录、恢复、退出和失效；它是 UI 与存储、网络之间的唯一边界。
- API 基址由 `BuildConfig` 暴露。Release 固定为 HTTPS 生产地址；Debug 可通过构建属性覆盖，且不允许提交账号、密码或 Token。
- 所有已认证请求通过统一的 Bearer Token 提供器添加 `Authorization: Bearer <token>`，不在业务 UI 中自行拼接请求头。

### 会话与安全存储

- 本地保存服务端 Token、公司标识、账号、显示名、角色和权限快照；保存范围仅限 12 小时会话恢复所需字段。
- 数据使用 Android Keystore 支持的加密键值存储；不保存用户密码，不将 Token 写入日志、异常文本或测试快照。
- 启动时先恢复本地会话：存在会话即进入应用壳层；不存在则进入登录页。服务端仍是 Token 有效性的最终裁决。
- 显式退出或收到任一已认证请求的 HTTP 401 时，立即清除本地会话，并切回登录状态。`SESSION_EXPIRED` 使用“登录已过期，请重新登录”提示；其他 401 使用“登录状态无效，请重新登录”提示。

### 领域模型与权限映射

- 新增独立的网络 DTO 与持久化会话记录，避免将服务器 JSON 直接暴露给 Compose UI。
- 将服务器 `admin` 映射为 `UserRole.ADMINISTRATOR`，`staff` 映射为 `UserRole.EMPLOYEE`。
- 将服务端权限键映射到现有 `AppPermission`。服务端管理员拥有全量权限；员工仅得到接口返回的权限，缺失或未知权限一律不放行。
- `AppSession` 扩展为包含真实会话所需的公司 ID、账号和 Token，但现有工作台继续只通过 `PermissionSnapshot` 决定可执行动作。

### UI 与导航

- 新增认证状态根入口：恢复中显示无敏感信息的加载状态；未认证显示登录页；已认证显示现有 `AutoserviceShell`。
- 登录页包含公司下拉选择、账号、密码、登录按钮、网络/服务器错误提示与提交中状态。仅支持账号密码，不展示访问码字段。
- “我的”页新增退出登录动作；完成退出后回到登录页并清空密码输入。
- 离线时登录请求不可发出，并给出网络不可用提示。已登录用户仍遵循现有只读模式。

## 数据流与失败处理

1. 用户提交公司、账号和密码。
2. `AuthRepository` 调用 `POST /api/access`，将成功响应映射、加密保存，并发布已认证状态。
3. 根入口切换到现有五栏壳层；工作台订阅新的真实会话并立即按权限重算。
4. 任一已认证 API 调用附带 Bearer Token。网络失败保留本地会话和当前页面；用户可重试。
5. 统一网络层检测到 HTTP 401 后通知 `AuthRepository` 清除会话；根入口切回登录页。
6. 用户从“我的”退出时，只清除本地会话；服务端现有契约没有注销端点，因此不增加伪造的远程注销请求。

## 测试与验收

- JVM 单测：请求/响应映射、角色和权限映射、未知权限拒绝、会话恢复、登录成功/失败、401 清除、退出清除与不持久化密码。
- Android 测试代码：登录入口和退出控件的可编译性，以及根入口在认证状态间切换的覆盖。
- 构建验证：JVM 单测、Android 测试代码编译、Lint、Debug APK 构建；不启动模拟器。
- 产物：将 Debug APK 复制到 `dist/releases/android/`，用于 API 26+ 真机安装测试。
