# Dashboard 21st Redesign Design

## Goal

将首页改造成浅色科技风的门店经营驾驶舱，让今日经营、待办风险、维修状态和最近工单在首屏内清晰可见，同时保留现有实时数据、筛选和跳转逻辑。

## Visual Direction

- 采用 21st `Statistics Card 1` 的指标层级：小标题、醒目数值、底部分隔说明。
- 采用 21st `Task List` 的整行任务操作：风险色只用于左侧标识和数量，不铺满卡片。
- 采用 21st `Table Dialog` 的固定表头、独立滚动、状态徽标和整行点击。
- 主色为业务蓝，背景为雾蓝灰；橙色表示待结算，红色表示保险风险，绿色表示正常或已结算。
- 卡片圆角统一 8px，使用轻边框和环境色阴影，不使用装饰性大图和嵌套卡片。

## Information Architecture

1. 顶部经营摘要：页面标题、统计范围、刷新状态、云端刷新按钮。
2. 核心指标：今日产值、今日台次、待结算金额、保险到期、本月/筛选产值。
3. 工作区：左侧产值趋势与维修状态，右侧优先待办。
4. 维修状态条：在修、完工、待结算、已结算，显示数量与最近一辆车。
5. 最近工单：固定表头、独立滚动、状态筛选和行点击。
6. 保险提醒：保留在首屏下方，并与最近工单形成辅助信息区。

## Behavior

- 指标卡和状态卡点击后进入对应维修或保险列表。
- 待办项整行可点击；待结算存在时显示下一笔工单。
- 最近工单表格保持状态筛选、分页和查看详情能力。
- 所有数据继续使用 `filteredOrders`、`policies` 和现有日期范围，不增加假数据。

## Responsive Rules

- 大屏为五列指标、主区 2:1、状态四列。
- 中屏指标自动降为三列，主区改为单列。
- 小屏指标单列，表格保留横向滚动，按钮和文本不溢出。

## Verification

- `npm.cmd run build` 必须通过。
- 在浏览器检查 1440x900 和移动宽度，确认无横向页面溢出、文字遮挡或空白巨型面板。
- 验证刷新、指标跳转、待办跳转、状态跳转和工单行点击。
