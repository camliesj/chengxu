# Android 工单读取与 Room 缓存设计

## 背景

Android 工作台当前通过 `DemoWorkbenchRepository` 展示两条硬编码工单，员工和管理员指标也使用固定演示数字。认证与加密会话已经接入生产服务端，下一阶段需要使用当前会话调用 `GET /api/orders`，同时继续满足既有的离线只读与公司数据隔离约束。

本阶段只实现工单读取、缓存、同步状态和工作台展示，不实现新增、编辑、删除、结算或状态推进。

## 目标

- 登录后立即读取当前公司的本地工单缓存，并在联网时刷新生产数据。
- 刷新成功后原子替换当前公司的缓存，避免混合展示新旧批次。
- 离线或刷新失败时继续展示已有缓存，并明确提示数据可能不是最新。
- 会话失效、退出登录或账号/公司切换时清除工单缓存，防止客户资料跨账号残留。
- 工作台最近工单、员工指标和管理员指标全部由真实工单计算，不再保留演示数据。
- Token 只从当前内存会话读取，不写入 Room、不写日志。

## 架构

### 远端边界

新增 `OrdersApi`，使用当前会话 Token 发送带 Bearer 认证的 `GET /api/orders` 请求。远端结果显式区分：

- 成功：返回经过容错解析的工单列表；
- 未授权或会话过期：触发会话失效处理；
- 网络、服务器或响应格式失败：保留缓存并暴露可读同步错误；
- 协程取消：原样向上传播，不转换为业务错误。

HTTP 实现沿用认证客户端的 API Origin、超时和无敏感日志约束。

### 本地缓存

新增 Room 数据库、`OrderEntity` 和 DAO。工单以 `companyId + orderId` 作为复合主键，并至少保存本阶段展示与指标所需字段：日期、时间、车牌、客户、车型、维修类型、状态、金额、维修记录、保险到期日和预计交车信息。

DAO 按公司观察数据，并以日期、时间和稳定 ID 倒序返回。一次成功刷新必须在单个事务中删除并插入当前公司的数据；其他公司的数据不得被查询或覆盖。

### 仓库与装配

生产工单仓库负责组合当前会话、网络状态、远端 API 和 Room DAO：

1. 会话建立后立即订阅当前公司的缓存；
2. 仅在网络在线且存在有效会话时请求远端；
3. 使用互斥锁合并启动、恢复会话和网络恢复等重复刷新；
4. 刷新成功后事务替换缓存，界面由 Room Flow 自动更新；
5. 网络恢复后自动重试；界面也提供显式重试入口；
6. 401 通过窄化的会话失效接口通知认证层，随后清除缓存并回到登录页；
7. 主动退出、认证失效、账号或公司切换时清除全部工单缓存。

`MainActivity` 改为装配生产 Room 数据库和工单仓库，不再注入 `DemoWorkbenchRepository`。

## 数据映射与业务口径

服务端状态统一使用 `在修中`、`已完工`、`待结算`、`已结算`。未知状态保留原文本并使用中性视觉样式，不能导致页面崩溃。

最近工单按日期、时间倒序展示。维修摘要优先使用维修记录；缺失时回退为车型与维修类型。金额和日期解析失败时使用安全默认值并保留该条工单。

员工指标口径：

- 今日接车：工单日期等于设备本地今天；
- 在修车辆：状态为 `在修中`；
- 待交付：状态为 `已完工`；
- 保险到期：到期日已经逾期或处于未来 3 天内。

管理员指标口径：

- 本月产值：本月全部工单金额之和；
- 待结算金额：状态为 `待结算` 的金额之和，并显示工单数；
- 在修车辆：状态为 `在修中`；
- 保险到期：到期日已经逾期或处于未来 7 天内。

## 界面状态

工单仓库公开缓存内容与独立同步状态，工作台据此呈现：

- 首次读取缓存时显示加载状态；
- 有缓存且刷新中时继续展示内容，并显示轻量同步提示；
- 有缓存但刷新失败时继续展示缓存，并提示“数据可能不是最新”；
- 无缓存且刷新失败时展示空状态、错误说明和重试入口；
- 离线时不发起请求，沿用全局“网络不可用，当前为只读模式”提示；
- 401 或会话过期时不显示旧缓存，直接回到登录流程。

## 测试策略

遵循测试驱动，先写失败测试，再补最小实现：

1. 远端 JSON 映射测试：成功、401、网络错误、异常状态码、畸形字段和协程取消；
2. 仓库测试：先读缓存、在线刷新、离线不请求、失败保留缓存、成功原子替换、公司隔离、重复刷新合并和 401 会话失效；
3. 指标与展示映射测试：日期边界、状态口径、金额合计、保险 3/7 天窗口、倒序和回退文案；
4. Room Android 测试：按公司查询、事务替换、排序和清除；保留测试代码并执行 Android 测试代码编译，不启动模拟器；
5. 回归执行 JVM 单元测试、Android 测试代码编译、`lintDebug` 和 Debug APK 构建。

功能完成后将可安装 APK 复制到 `dist/releases/android/`，记录大小与 SHA-256，交由用户在真实手机上验证。

## 非目标

- 不修改服务端 `/api/orders` 契约或网页端工单界面；
- 不实现工单新增、编辑、删除、状态推进、结算或离线写入队列；
- 不跨登录保留任何其他公司或上一账号的客户数据；
- 不在本阶段引入后台定时同步、分页或复杂缓存淘汰策略。
