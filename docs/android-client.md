# Android 客户端交付与真机测试

## 环境要求

- JDK 17。
- Android SDK Platform 36；构建目标为 `compileSdk 36`、`targetSdk 35`、`minSdk 26`。
- 一台 API 26 及以上的 Android 真机用于安装测试；本阶段不要求启动模拟器。
- 在仓库根目录 `E:\codex\chengxu` 执行 Git 命令，在 `android-client\` 执行 Gradle 命令。

## 首次构建

```powershell
$env:JAVA_HOME='E:\codex\APP\.android-build\jdk\jdk-17.0.19+10'
$env:ANDROID_HOME='E:\codex\APP\.android-build\android-sdk'
cd E:\codex\chengxu\android-client
.\gradlew.bat :app:assembleDebug
```

构建输出为 `app\build\outputs\apk\debug\app-debug.apk`。当前交付副本为 `E:\codex\chengxu\dist\releases\android\autoservice-android-debug-0.1.0.apk`。

## 运行测试

```powershell
$env:JAVA_HOME='E:\codex\APP\.android-build\jdk\jdk-17.0.19+10'
$env:ANDROID_HOME='E:\codex\APP\.android-build\android-sdk'
cd E:\codex\chengxu\android-client
.\gradlew.bat clean :app:testDebugUnitTest :app:compileDebugAndroidTestKotlin :app:lintDebug :app:assembleDebug
```

`testDebugUnitTest` 运行 JVM 认证、加密会话、状态、权限、导航和网络门禁契约；`compileDebugAndroidTestKotlin` 仅验证 Compose/Activity 测试代码可编译，不会启动设备或模拟器。`lintDebug` 执行 Android 静态检查。

登录与 AndroidKeyStore 相关改动还应在已连接设备上执行：

```powershell
.\gradlew.bat :app:connectedDebugAndroidTest
```

## 品牌 UI 真机验收

请在 API 26 及以上真实 Android 设备安装当前 APK，并按顺序检查：

1. 登录页同时显示通达和鑫齐恒两张 56dp 企业选择卡，只展示短业务名称；点击后冰蓝容器、近黑边框、勾选图标和选中语义同步切换。
2. 在约 360dp 宽、800dp 高的竖屏设备上，关闭输入法后确认无需滚动即可看到完整“进入系统”按钮；200dp 冰蓝 Hero、两行标题和车辆均清晰，白色面板只轻微覆盖车辆底部。
3. 点击账号或密码弹出输入法，确认 Hero 收拢为 96dp 并隐藏车辆，当前字段和“进入系统”仍可到达；横屏、放大系统字体等确实无法容纳内容时仍可纵向滚动。
4. 账号、密码输入、显示/隐藏密码、软键盘 Done 提交、空字段/错误凭据、提交中禁用和真实登录均正常；盾牌安全说明在常见 390×844dp 视口中完整显示。
5. 分别使用真实员工和管理员账号登录；员工显示“今日概览/我的待办”且没有“办理结算”，管理员显示“经营概览/优先事项”并具有服务端授权的结算操作。
6. 底部始终按“工作台 / 工单 / 新增 / 档案 / 我的”排列；中间“新增”为近黑圆形主操作，五个标签切换和各自返回栈均正常，“工单”显示真实缓存列表而非阶段占位页。
7. 打开飞行模式或同时关闭 Wi-Fi/移动数据；顶部显示“网络不可用，当前为只读模式”，浏览仍可进行，中间“新增”禁用，其他写操作显示同一门禁原因且不产生伪写入。
8. “我的”显示真实姓名、企业、角色、同步与本机加密状态；第一次点击退出只打开确认弹窗，“暂不退出”保留当前登录，确认退出后返回品牌登录页。
9. 在约 360dp 宽度的设备上检查四列状态带、两列指标、快捷操作、工单卡、系统状态栏/导航栏和软键盘场景；不得出现横向裁切、遮挡或不可点击控件。

## 真实登录与会话真机检查

当前客户端调用 `https://chengxu.pages.dev/api/access`，只接受“公司 + 账号 + 密码”。两张企业选择卡固定为“通达汽车服务中心”和“鑫齐恒汽车服务中心”；密码和 Token 不写入日志，密码不持久化。

请在真机依次验证：

1. 选择通达，使用有效账号密码登录；确认进入工作台，“我的”显示服务端返回的姓名、公司和角色。
2. 退出登录，再选择鑫齐恒并使用有效账号密码登录；确认没有短暂显示上一账号的姓名、公司或权限界面。
3. 输入错误密码；确认停留在登录页并显示“账号、密码或公司不正确”，不会进入五栏壳层。
4. 成功登录后强制停止并重新打开应用；在登录后 12 小时内应从加密本地会话直接恢复，不要求重新输入密码。
5. 在“我的”点击“退出登录”；确认先出现退出确认弹窗，点击“暂不退出”仍留在当前会话，再次打开弹窗并确认后回到登录页。强制停止并重新打开后仍应停留在登录页。
6. 保留登录超过服务端 12 小时有效期，或让已认证工单接口返回 `SESSION_EXPIRED`，确认工单缓存先清除，本地会话随后失效，并提示“登录已过期，请重新登录”。

## 真实工单与缓存真机验收

1. 在线登录通达账号，确认工作台先显示该公司 Room 缓存，再出现“正在同步…”并更新为生产 `/api/orders` 数据；员工指标与最近工单随真实数据变化。
2. 在线登录鑫齐恒账号，确认不会闪现通达的工单、客户、车牌或指标；同公司切换账号也不得残留上一账号缓存。
3. 在线同步成功后强制停止并重新打开应用，确认加密会话恢复后立即显示缓存；随后联网刷新，已有卡片在刷新过程中保持可见。
4. 有缓存时打开飞行模式，确认继续显示缓存、顶部进入只读模式且不发出工单请求；恢复网络后自动刷新并移除只读横幅。
5. 退出登录清空缓存后，在离线状态重新打开应用，确认未认证时停留登录页；若以测试手段进入空缓存离线状态，工作台显示“暂无工单数据”且不产生写入入口。
6. 模拟网络错误、服务端 5xx 或异常工单响应，确认已有卡片保留并显示对应提示；点击“重新同步”只发起一次刷新。空缓存失败时仍显示错误和重试按钮。
7. 让工单接口返回 401，确认缓存被清除并返回登录页；重新进入后不得看到失效会话的客户数据。
8. 分别核对员工与管理员真实指标：今日接车、在修中、已完工待交付、本月产值、待结算金额/单数，以及 3 日/7 日保险到期窗口；金额千分位与小数应正确。
9. 确认本里程碑没有新增创建、编辑、删除、结算、状态推进、离线写队列或其他真实写入能力，既有权限与离线门禁保持不变。
10. 打开“工单”，核对页头总数、卡片顺序与生产接口/工作台一致；“在修中 / 已完工 / 待结算 / 已结算”状态芯片颜色与文案正确，服务端未知状态保留原文且只出现在“全部”。
11. 分别输入完整或部分工单号、车牌、客户、车型和维修记录，确认本地搜索命中正确且不会发起新的服务端搜索请求；清空输入后恢复原顺序。
12. 依次选择“全部 / 在修中 / 已完工 / 待结算 / 已结算”，确认筛选与搜索取交集；没有结果时显示“没有匹配的工单”，点击“清除筛选”一次即可恢复全部。
13. 在线刷新时确认已有卡片保持可见并显示“正在同步…”；制造 5xx/异常响应后确认缓存保留、陈旧提示出现，点击“重新同步”只触发一次刷新。
14. 有缓存时断网，确认列表和详情仍可阅读、顶部显示只读横幅且陈旧面板不提供“重新同步”；恢复已验证网络后才允许刷新。
15. 分别从“工单”卡片和工作台最近工单打开同一 ID，核对工单号、日期时间、车牌、客户、车型、维修类型、维修记录、状态、总金额、保险到期和预计交车；两个根标签各自保留详情返回栈。
16. 打开详情后让下一次同步移除该工单，确认当前页切换为“工单不存在或已失效”，点击“返回工单列表”回到对应根页，不继续展示旧客户数据。
17. 列表和详情不得出现新增、编辑、删除、状态推进、结算、作废、回执或导出控件；所有卡片、筛选、返回、重试和清除筛选触控高度至少 48dp。
18. 在约 360dp 宽、系统大字体、横屏和输入法弹出场景检查搜索框、横向滚动筛选条、长工单号/车牌/客户、四个详情分区与底栏；不得横向裁切，筛选条必须可滚动，内容必须可纵向到达。

## 阶段 1 生产数据基础真机回归

1. 覆盖安装当前 APK，使用通达账号登录并确认当前工单列表、详情、搜索、筛选和离线只读行为与上一版一致；本阶段没有新增任何写入按钮或页面。
2. 再退出并使用鑫齐恒账号登录，确认两家公司的摘要、详情、草稿占位、同步游标和档案基础数据均隔离，不能闪现另一公司的客户、手机号、VIN、车辆或保险信息。
3. 在线刷新后断网并重启应用，确认仍可读取当前公司的 Room 摘要缓存；网络恢复后 legacy `/api/orders` 正常刷新，列表不因新增扩展 API 而改变。
4. 退出登录后再次打开应用，确认摘要、详情、草稿、同步游标、车辆和保险基础表均已清理，不显示上个会话的数据；让工单请求返回 401 时重复相同检查。
5. 在保留 v1 数据库的旧版本上先同步至少一条工单，再覆盖安装本 APK；登录后确认旧工单仍可读取，应用不因 Room v1→v2 显式迁移崩溃或清空既有摘要。
6. 对包含手机号/VIN 的详情和包含手机号的草稿执行后续开发测试时，确认 Room 只出现加密字段/载荷；回执对象 key、二进制和短期 URL 不得写入本地数据库。
7. 说明：阶段 1 仅交付生产数据/API/加密/Room 基础，扩展 current/history/detail 读取尚未替换现有 UI 仓库，新增、编辑、状态推进、结算、作废、回执维护和导出入口均尚未上线。

## 断网验证

1. 退出登录，在真机关闭 Wi-Fi 与移动数据，填写账号密码并点击登录；确认不发送登录请求并显示“网络不可用，请检查网络连接后重试”。
2. 已登录时关闭网络，等待页面顶部出现“网络不可用，当前为只读模式”。
3. 确认底部第三项“新增”不可点击。
4. 恢复网络，确认横幅会在网络通过系统验证后消失；未验证的热点仍必须保持只读。

离线时所有写入动作由 `MutationGate` 拒绝，拒绝提示使用同一只读文案。
