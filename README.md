# 汽修接待与车辆保险管理软件

这是一个用于汽修门店的本地管理软件项目，目标是实现客户与车辆档案、维修接待工单、车辆保险信息、查询筛选、月度汇总、可视化统计和 Excel 导出。

## 当前状态

项目已配置本地 Git 接力基础。后续正式开发时，会在这里补充具体的启动、打包和数据维护命令。

## 接力开发方式

1. 在当前电脑开发时，每完成一个小功能就提交一次。
2. 换电脑继续开发前，确认当前电脑已经提交所有代码改动。
3. 如果已经绑定 GitHub、Gitee 或 GitLab 远程仓库，先推送最新代码。
4. 在新电脑拉取仓库代码，安装依赖后继续开发。
5. 如果还没有远程仓库，可以先整体复制项目文件夹，但必须保留隐藏的 `.git` 目录。

## 后续绑定远程仓库

拿到远程仓库地址后，在项目目录执行：

```bash
git remote add origin <仓库地址>
git push -u origin main
```

常见选择：

- GitHub 私有仓库
- Gitee 私有仓库
- GitLab 私有仓库

## 本地数据与备份

计划使用本地 SQLite 数据库保存业务数据。数据库文件属于本机运行数据，不直接提交到 Git。

推荐规则：

- 代码通过 Git 接力。
- 数据库文件单独备份或复制。
- 真实配置文件 `.env` 不提交，只提交 `.env.example` 作为模板。
- 需要演示数据时，后续单独准备可导入的示例数据文件。

默认本地数据库路径预留为：

```text
./data/app.sqlite
```

## Excel 导出

后续开发会支持导出：

- 维修工单明细
- 保险信息明细
- 月度汇总表

导出文件属于运行产物，不提交到 Git。

## 计划中的核心功能

- 客户信息录入与查询
- 车辆信息录入与查询
- 维修接待工单管理
- 工时项目和领料项目费用录入
- 维修状态切换：在修、完工、结算
- 保险公司送修信息记录
- 三者车/标的车记录
- 车辆保险到期、险种、金额维护
- 日期范围统计与可视化
- 月度产值、台次、费用汇总
- Excel 导出
